# Links 用户指南

> 你的本地链接收藏夹。无需账号，无需联网，所有数据只属于你。

---

## 设计理念

Links 的核心想法很简单：**收藏一个链接，应该像粘贴一样容易，找回来应该像搜索一样快。**

我们做了几个设计选择：

- **本地优先** — 所有数据存储在你电脑上的 SQLite 数据库中，不需要注册账号，不需要云服务同步。你的书签不会被任何人看到。
- **轻量安静** — 安装包约 5MB，内存占用极低。平时缩在系统托盘里不碍事，需要时一键唤出。
- **自动抓取** — 粘贴 URL 后自动获取标题、描述和图标，减少手动填写。如果不小心关掉了，链接已经存好了。
- **中文友好** — 内置基于 FTS5 的全文搜索引擎，中文搜索无压力。

## 快速上手

### 安装

| 平台 | 安装包 |
|------|--------|
| Windows | 下载 `.exe` 安装程序或 `.msi` |
| macOS | 下载 `.dmg`，拖入应用程序文件夹 |
| Linux | 下载 `.deb` 或 `.AppImage` |

### 第一次使用

1. 安装后启动 Links，你会看到一个简洁的界面：左侧是侧边栏，右侧是链接列表。
2. 点击右下角的 **+** 按钮添加你的第一个链接。
3. 粘贴 URL 后，标题和描述会自动填充 — 直接点保存就好。
4. 在日常使用中，探索Links的其他功能。

**强烈推荐**：从浏览器导出你的所有收藏，导入到Links（未来支持自动将收藏夹映射到分组上）

### 全局快捷键

不用先打开主窗口，随时随地保存链接：

| 快捷键 | 功能 | 默认按键 |
|--------|------|----------|
| 快速添加 | 唤出精简的添加窗口 | `Cmd+Shift+L`（macOS）/ `Ctrl+Shift+L`（Windows） |
| 主窗口 | 显示并聚焦主窗口 | `Cmd+Shift+J`（macOS）/ `Ctrl+Shift+J`（Windows） |

快捷键可以在 **设置 → 快捷键** 中自定义。

## 核心功能

### 添加链接

#### 通过主窗口添加

1. 点击右下角 **+** 按钮，弹出添加表单。
2. 填写内容：

| 字段 | 必填 | 说明 |
|------|------|------|
| URL | ✅ | 链接地址，以 `http://` 或 `https://` 开头 |
| 标题 | — | 粘贴 URL 后自动抓取，也可手动修改 |
| 分组 | — | 将链接归入某个分组 |
| 标签 | — | 添加多个标签，方便后续筛选 |
| 描述 | — | 页面描述，通常自动抓取 |
| 备注 | — | 你自己写的想法 |

3. 粘贴 URL 后，Links 会自动：
   - **抓取元数据**（标题、描述、网站图标、封面图）
   - **检测重复**（如果已收藏过相同 URL，会提示警告）

4. 点击保存完成添加。

#### 通过快速添加窗口

按下 `Cmd+Shift+L`（macOS）或 `Ctrl+Shift+L`（Windows），一个精简的添加窗口会浮现在屏幕最前方。粘贴 URL → 保存 → 窗口自动关闭。整个过程不用离开当前工作。

快速添加窗口的特点：
- 始终置顶显示，不会挡住你的工作流程
- 没有任务栏图标，干净不干扰
- 每次唤出时自动清空上次的内容，并聚焦到 URL 输入框
- 按 `Esc` 可以关闭窗口
- 窗口失焦后自动关闭

#### 编辑链接

在链接卡片上点击编辑按钮（铅笔图标），可以修改链接的所有信息。也可以点击 URL 输入框旁的刷新按钮重新抓取元数据。

### 管理链接

#### 收藏/取消收藏

点击链接卡片右侧的书签按钮（旗帜图标）即可切换收藏状态。已收藏的链接会在卡片右上角显示一个金色书签标记。

#### 删除链接

点击删除按钮（垃圾桶图标）后，会弹出确认提示："确定要和这个链接说再见吗？"。点击"没错"确认删除，点击"算了"取消。

#### 分享链接

点击分享按钮（链接图标），可以将链接以三种格式复制到剪贴板：

| 格式 | 用途 | 示例 |
|------|------|------|
| 复制链接 | 直接粘贴 URL | `https://example.com` |
| 复制 Markdown | 粘贴到 Markdown 文档 | `[示例网站](https://example.com)` |
| 复制 HTML | 粘贴到网页 | `<a href="...">示例网站</a>` |

复制成功后会显示"已复制!"提示。

#### 失效链接检测

添加链接后，Links 会自动检测链接的可达性。如果链接无法访问，卡片上会显示一个黄色的警告图标，提示"链接可能已失效"。

### 搜索

#### 全文搜索

顶部搜索栏支持实时搜索，输入关键词即可。搜索范围包括：

- 链接标题
- 链接描述
- 链接备注
- 标签名称
- 链接 URL

一言蔽之，就是所有信息都可以搜索

搜索结果会**高亮显示**匹配的关键词。

搜索使用 SQLite FTS5（全文搜索引擎），对中文有良好的支持。如果 FTS5 搜索没有结果，会自动降级为模糊匹配，确保不遗漏。

#### 按分组筛选

在左侧侧边栏点击某个分组，右侧只显示该分组下的链接。

#### 按标签筛选

在左侧标签列表中点击某个标签，右侧只显示带有该标签的链接。

#### 特殊筛选

侧边栏提供了两个特殊筛选选项：

| 筛选项 | 说明 |
|--------|------|
| 未分组 | 显示没有归入任何分组的链接 |
| 无标签 | 显示没有添加任何标签的链接 |

### 分组管理

分组支持树形嵌套结构，可以创建多级分类。（预计下个版本推出）

#### 创建分组

1. 在侧边栏"分组"标题旁点击 **+** 按钮。
2. 输入分组名称，按回车确认。
3. 可以选择将新分组放在某个现有分组之下（作为子分组）。

#### 展开/收缩

点击分组名称左侧的箭头图标可以展开或收缩子分组。

#### 重命名分组

点击分组名称右侧的编辑图标，输入新名称后按回车或点击其他区域确认。

#### 删除分组

点击分组左侧的删除图标。第一次点击会显示"再点一下就删除"提示，再次点击才真正删除。鼠标移开后恢复原状，防止误操作。

**注意**：删除分组不会删除其中的链接，链接只是变成了"未分组"状态。

### 标签管理

#### 添加标签

- 添加/编辑链接时，在标签输入框中输入标签名称。
- 输入时会自动弹出已有标签的补全建议。
- 如果输入的标签不存在，可以直接创建新标签。
- 如果标签数量超过 10 个，标签列表上方会出现搜索框。

#### 重命名标签

点击标签名称右侧的编辑图标，输入新名称后按回车确认。重命名标签会影响所有使用该标签的链接。

#### 删除标签

和分组一样采用两次点击确认机制：第一次点击显示"再点一下就删除"提示，再次点击确认删除。

**注意**：删除标签会从所有使用该标签的链接上移除，但不会删除链接本身。

#### 从链接上移除标签/分组

在链接卡片上，将鼠标悬停在标签或分组的小图标上，点击出现的删除图标即可移除。移除后链接仍然存在，只是不再关联该标签或分组。

### 导入书签

Links 支持导入浏览器的书签文件：

1. 从浏览器导出书签为 HTML 文件：
   - **Chrome**：书签管理器 → 右上角菜单 → 导出书签
   - **Firefox**：书签 → 管理所有书签 → 导入和备份 → 导出为 HTML
   - **Safari**：文件 → 导出书签
2. 在 Links 侧边栏底部点击导入按钮。
3. 选择导出的 HTML 文件，Links 会自动解析并导入所有书签。

### 导出链接

Links 支持将链接导出为三种格式：

| 格式 | 扩展名 | 适用场景 |
|------|--------|----------|
| JSON | `.json` | 结构化数据，可以用程序处理，也可以重新导入 |
| Markdown | `.md` | 纯文本格式，方便阅读和分享 |
| CSV | `.csv` | 表格格式，可以在 Excel 或 Google Sheets 中打开 |

点击侧边栏底部的导出按钮，选择格式后点击导出，系统会弹出文件保存对话框让你选择保存位置。

## 界面一览

### 主窗口

```
┌─────────────────────────────────────────────┐
│  侧边栏  │        内容区                      │
│           │                                   │
│  🔗 Links │  [搜索栏: 找找看~]                 │
│           │                                   │
│  收藏夹   │  链接卡片 1                        │
│  未分组   │  链接卡片 2                        │
│           │  链接卡片 3                        │
│  分组     │  ...                              │
│   ├ 工作  │                                   │
│   └ 学习  │                                   │
│           │                              [+]  │
│  标签     │                                   │
│   rust    │                                   │
│   设计    │                                   │
│           │                                   │
│  ☀️ 🌙 ⚙️ │                                   │
└─────────────────────────────────────────────┘
```

- **侧边栏**：分组、标签、快捷筛选
- **内容区**：链接卡片列表，向下滚动自动加载更多
- **搜索栏**：输入即搜索，支持中文全文检索
- **+ 按钮**：右下角浮动按钮，点击添加新链接
- **侧边栏底部**：亮/暗模式切换、设置入口

### 链接卡片

每张卡片展示一条链接的信息：

- **标题行**：网站图标 + 标题。标题过长时鼠标悬停显示全文。
- **域名**：默认显示域名（如 `github.com`），鼠标悬停原地展开为完整 URL。
- **描述**：页面描述摘要，过长时悬停显示全文。
- **标签栏**：分组标签（带文件夹图标）、标签（带标签图标）、备注（带文档图标）。最多显示 5 个标签，超出部分显示 `+N`。悬停标签/分组图标可点击移除。
- **操作按钮**（悬停显示）：收藏、编辑、分享、删除
- **书签标记**：已收藏的链接右上角显示金色书签，悬停提示"取消标记"
- **失效标记**：链接不可达时显示黄色警告图标

## 个性化设置

点击侧边栏底部的齿轮图标打开设置面板。

### 关闭行为

控制点击窗口关闭按钮（×）时发生什么：

| 选项 | 说明 |
|------|------|
| 每次询问 | 弹出对话框让你选择 |
| 最小化到托盘 | 关闭窗口后继续在后台运行 |
| 直接退出 | 关闭窗口即退出应用 |

如果你经常用全局快捷键添加链接，推荐选择"最小化到托盘"。

### 外观

| 选项 | 说明 |
|------|------|
| 亮色 | 始终使用亮色主题 |
| 暗色 | 始终使用暗色主题 |
| 跟随系统 | 自动匹配系统外观设置（默认） |

切换主题时不会闪烁，快速添加窗口也会同步更新。

### 开机自启动

开启后，系统启动时 Links 会自动在后台运行。

### 快捷键

可以分别自定义两个全局快捷键：

| 快捷键 | 默认 | 功能 |
|--------|------|------|
| 快速添加 | `Cmd+Shift+L` / `Ctrl+Shift+L` | 唤出快速添加窗口 |
| 主窗口 | `Cmd+Shift+J` / `Ctrl+Shift+J` | 唤起主窗口并聚焦搜索框 |

修改快捷键：点击"修改"按钮 → 按下你想设置的新组合键 → 点击"保存"。

快捷键格式要求：至少包含一个修饰键（Cmd/Ctrl、Shift、Alt）加上一个普通按键。

### 键盘快捷操作

| 按键 | 场景 | 效果 |
|------|------|------|
| `Esc` | 设置面板 | 关闭设置 |
| `Esc` | 导出面板 | 关闭导出 |
| `Esc` | 添加/编辑链接 | 取消并关闭表单 |
| `Esc` | 快速添加窗口 | 关闭窗口 |
| `Esc` | 关闭确认对话框 | 取消并关闭 |
| `Enter` | 标签/分组重命名 | 确认重命名 |
| `Enter` | 创建分组/标签 | 确认创建 |

## 数据与隐私

### 数据存储位置

所有数据存储在本地，不经过任何网络传输：

| 平台 | 路径 |
|------|------|
| macOS | `~/Library/Application Support/com.links.desktop/` |
| Windows | `%APPDATA%\com.links.desktop\` |
| Linux | `~/.local/share/com.links.desktop/` |

存储文件说明：

| 文件 | 说明 |
|------|------|
| `links.db` | SQLite 数据库，存储所有链接、分组、标签数据 |
| `config.json` | 配置文件，窗口大小/位置、主题、快捷键等偏好设置 |
| `links.log` | 应用日志，用于排查问题 |
| `fail_links.log` | 元数据抓取失败的链接记录 |

### 数据安全

- **不联网**：Links 不连接任何云服务。唯一的网络请求是你添加链接时抓取页面元数据。
- **不追踪**：没有分析工具、没有遥测、没有任何用户行为跟踪。
- **你的电脑就是服务器**：数据只存在于你的设备上。

### 数据库损坏恢复

如果数据库文件损坏（极少发生），Links 会在下次启动时自动重建空数据库。你可以从之前导出的 JSON 文件中恢复数据（需要重新导入）。

**建议**：定期导出链接作为备份。

### 卸载

卸载 Links 不会自动删除数据文件。如果你想彻底清除所有数据，需要手动删除上述数据目录。

## 常见问题

### 快捷键不生效

- 检查是否有其他应用占用了相同的快捷键组合。
- 在设置中尝试更换为其他组合键。
- 确认系统辅助功能权限已授予（macOS 可能需要在"系统设置 → 隐私与安全性 → 辅助功能"中允许 Links）。

### 元数据抓取失败

部分网站不允许自动抓取，或者需要登录才能访问。此时 Links 会提示"这个小站不想被抓取呢，手动填写信息吧"。你可以：

- 手动填写标题和描述
- 点击 URL 输入框旁的刷新按钮重试

### 端口 1420 被占用

开发模式下，Links 的前端服务使用端口 1420。如果该端口被其他程序占用，启动会报错。使用 `lsof -i :1420` 查看并关闭占用端口的程序。

### 窗口位置异常

如果窗口跑到了屏幕外面（连接/断开外接显示器后可能出现），Links 会在下次启动时自动检测，如果窗口不在任何屏幕范围内，会将其移回默认位置。

### 搜索不到某个链接

Links 使用 FTS5 全文搜索，对中文有良好支持。如果搜索不到，可能是：

- 链接是最近添加的，搜索索引尚未更新
- 搜索词与实际内容不完全匹配，尝试减少关键词

## 小提示

- 在链接卡片上**点击标题或描述区域**可以直接在浏览器中打开链接。
- 标签和分组的删除都需要**点击两次**确认，第一次只是提示，鼠标移开就恢复，不用担心误删。
- 域名**鼠标悬停**会展开为完整 URL，方便确认链接目标。
- 收藏的链接**右上角有金色书签**，一眼就能认出来。
- 添加链接时如果**手动修改了标题或描述**，后续的自动抓取不会再覆盖你的修改。
- 快速添加窗口**失焦后自动关闭**，保存好了就切走，不用手动关。
- 搜索栏支持**实时搜索**，输入 250ms 后自动触发，不用按回车。
